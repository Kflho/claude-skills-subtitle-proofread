# 参考字幕的使用

> 适用条件：用户提供了参考字幕目录（有高质量人工翻译字幕）。
> 参考字幕用于按时间码定位对照，提升 AI 校对质量。

## 核心原则

- 逐行对比同一时间码的目标文本与参考文本
- 发现不一致时，以参考字幕的语义为准修正目标文本
- 参考字幕中的专有名词（人名、地名、称谓）统一采纳为全剧标准

## 参考字幕在 Pipeline 中的位置

### Phase 2：AI 碎片补全

参考字幕中的原文被注入到 `ai_fragments_{EP}.json` 的 `reference_text` 字段。
AI 读原文 + 上下文 → 直接写日文 correction。参考字幕可能有错，需结合上下文判断。

### Phase 3：OP/ED 歌词统一

```
资源优先级：
Tier 1: 有参考字幕 → 提取 OP/ED 文本 → 自动填入 canonical，AI 验证
Tier 2: 有音频(Whisper) → 跨集对比 → AI 审查变体
Tier 3: 什么都没有 → 跨集对比 → AI 审查（默认）
底限:   人工审查
```

### Phase 3：专有名词统一

参考字幕中的专名可帮助验证 spelling variants 和发现机翻幻觉。

## 完整模式工作流

### 阶段一：规则批量修复

1. **统一扫描** — `unified_scanner.py`：乱码检测 + 重复检测 + 术语收集
2. **ASS 格式修补** — `ass_repair.py --check all`（ASS only）
3. **繁体→简体** — `trad_to_simp_detect.py --auto`（仅中文）
4. **感叹词替换** — `interjection_detect.py`（翻译项目）
5. **专有名词统一** — `noun_checker.py`（有参考字幕时更准确）
6. **机翻幻觉检测** — `garbled_detect.py --lang <LANG>`（中文）
7. **固定格式统一** — `format_detect.py --lang <LANG>`（翻译项目）

### 阶段二：批量精读审查

1. 将文件分为 N 批（每批 ~18 集）
2. 每批启动一个子代理，对照参考字幕逐行精读
3. 子代理输出 `OLD → NEW` 修复列表
4. 主进程统一审查：消除矛盾建议、合并重叠修复、确认译名一致性
5. 生成 fixes.json，运行 `apply_fixes.py` 一次性应用

### 阶段三：残留检查

重新运行关键检测脚本，确保无遗漏。

## 完整模式专属错误类型

### 音译代替意译
源语言感叹词/拟声词被音译而非翻译。对照参考字幕同一时间码确定正确语义。

### 机翻幻觉 / 假朋友
MT 将专有名词误译为字面意思。

| 类型 | 示例 |
|------|------|
| 人名→名人幻觉 | `Abe`→`安倍晋三`（应为角色名） |
| 假朋友 | `sister`→`修女` 而非 `姐姐` |
| 脏话误译 | "Serves you right!"→`去你妈的` |

### 节目名/角色名/术语不统一
运行 `noun_checker.py`，关注变体和拼写不一致。

### 源语言文本损坏
参考字幕可能存在字符级反转或编码损坏。对照参考字幕恢复后翻译。
